Rails on Heroku

Aug 15th, 2013 | Comments

I pushed bookmarks-rails project to Heroku yesterday. Thanks to tutorial i don’t have too many troblem during upload to Heroku. Some parts of doc are out of dated and here i write down the steps for reference myself in the future.

  1. install heroku toolbelt from heroku
  2. heroku doesn’t support sqllite3, modify sqllite3 to postgres sql

to

3.use thin as production webserver

4. run bundle install --without production
5. precompile your assets file, you have to add config.assets.initialize_on_precompile=false to your config/application.rb
6. run rake assets:precompile
7. run git commit -a -m "your commit message" because only commited source code will be pushed to heroku.
8. run heroku create appname to create your app on heroku. It requires you input your account,password and your ssh public key(id_rsa).
9. run heroku rake db:migrate to create db on heroku
10. run git push heroku master. You will see your app running on http://appname.herokuapp.com. Heroku will modify your production datbase connection in database.yml when push to heroku. You don’t need to put the production setting your database.yml.

trouble shooting

If you use Twitter Bootstrap, you have to comment out favicon related codes in your application.html.erb. Or you will fail to push to heroku.

If you modify the app name on Heroku web site, you should remove your remote branch and re-add.
